# Миграция фронтенда Hunter AI: монолит → React + Vite

Живой production SaaS. Backend (`api/`) НЕ трогается. Цель — перенести фронт из
`public/index.html` (~5500 строк ванильного JS+HTML+CSS) на React + Vite, 1:1 по виду и логике.

## ⚠️ Критично для безопасности прода
- **`vercel.json` НЕ менять до финала (Этап 8).** Сейчас `outputDirectory: "public"` — прод
  отдаёт старый монолит `public/index.html`. Если переключить сборку на `dist/` до готовности
  React-версии, следующий деплой Vercel выкатит недособранный фронт на живой бизнес.
- `public/index.html` НЕ удалять — на Этапе 8 переименовать в `public/index.legacy.html`.
- Форматы запросов/ответов к `/api/*` не менять.

## Запуск
- Node.js установлен в `C:\Program Files\nodejs` (v24). В PowerShell добавляй его в PATH:
  `$env:Path = "C:\Program Files\nodejs;$env:Path"` перед `npm`.
- `npm install` → `npm run dev` (порт 5173). Dev-прокси `/api/*` → прод Vercel.
- Укажи прод-URL в `.env.local`: `API_PROXY_TARGET=https://<твой-проект>.vercel.app`
  (по умолчанию `https://hunters-path.vercel.app`).
- `npm run build` → `dist/`.

## Архитектура переноса
- `src/lib/` — session (SESSION/role/org + localStorage `hp_session`), api (все обёртки
  `fetch('/api/...')`), i18n (`MOP_T`+`mt`), theme, format (`escapeHtml`/`fmtSumM`). Всё 1:1.
- `src/styles/global.css` — весь `<style>` монолита побайтово (темы, переменные, адаптив).
- **Паттерн переноса render-функций:** тяжёлые функции монолита, которые строят HTML-строки
  (`renderMopEarnings` и т.п.), переносятся **дословно** в `*.js` и вставляются через
  `dangerouslySetInnerHTML` — это гарантирует пиксельную идентичность. Интерактив (onclick)
  выносится в React-обработчики там, где он есть в разметке.
